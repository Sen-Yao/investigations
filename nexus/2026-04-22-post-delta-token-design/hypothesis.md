# Hypotheses

## H1
真正有效的 anomaly-aware token 不应只是 hop/delta 的重参数化，而应来自不同信息来源。

## H2
Community / Patch Token 能更直接承接 ANR 所揭示的社区型异常现象。

## H3
Prototype Assignment / Contrast Token 比 mean-delta prototype 更接近“模式归属”与“偏离参考”的异常检测语义。

## H4
Relation Token 能把异常检测中的“关系异常”显式 token 化，因此比单节点状态 token 更有新增量潜力。

## H5
Tokenphormer 对 VoxG 的真正启发是 multi-token family 的构造原则，而不是一阶差分这种具体 token 形式。
