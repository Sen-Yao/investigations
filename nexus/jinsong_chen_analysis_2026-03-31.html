📖 核心方法详解

1. NAGphormer (ICLR 2023)

核心问题

传统 Graph Transformer 将所有节点作为 token，导致 O(N²) 复杂度，无法扩展到大图。

核心创新：Hop2Token

对于每个节点 v:
  Token_0 = Feature(v)           # 自身特征
  Token_1 = Aggregate(Hop_1(v))  # 1-hop 邻居聚合
  Token_2 = Aggregate(Hop_2(v))  # 2-hop 邻居聚合
  ...
  Token_K = Aggregate(Hop_K(v))  # K-hop 邻居聚合
  
输入序列 = [Token_0, Token_1, ..., Token_K]  # 长度 K+1

关键特点

特点说明
可扩展性每个节点独立处理，支持 mini-batch 训练
线性复杂度O(N × K) 而非 O(N²)
多 hop 信息捕获不同距离的邻居信息
Attention Readout自适应聚合不同 hop 的信息

局限性

  • 每个 hop 的邻居聚合成单个 token，丢失了邻居的多样性信息
  • 不同 hop 之间独立，没有捕获 hop 间的变化趋势
  • 没有考虑差分信息（如 Delta、Offset）

2. NTFormer (IEEE TBD 2025)

核心问题

现有方法只使用单一类型的 token 生成，只能表达部分图信息，需要针对不同图设计特定策略。

核心创新：Node2Par

对于每个节点 v，生成多种 token 类型：

Token 类型 A: 节点自身特征
Token 类型 B: 邻居特征聚合
Token 类型 C: 属性特征
Token 类型 D: 结构特征 (度、PageRank 等)

组合生成多种 token 序列 → 全面表达图特征

关键特点

特点说明
多视角从不同角度表达节点特征
通用性无需针对不同图类型修改模型
灵活组合可根据任务选择合适的 token 类型

与 Hop-Offset 的关系

Offset 可作为 Node2Par 的一种 token 类型！
  • Token 类型 X: Offset 向量 (hop_k - hop_0)
  • 与其他 token 类型组合，提供差分视角

3. SwapGT (NeurIPS 2025)

核心问题

现有 tokenized GT 只关注一阶邻居，生成的 token 序列多样性不足。

核心创新：Token Swapping

利用节点间的语义相关性：

如果 node_i 和 node_j 语义相似（如在同一类中）：
  可以交换它们某些位置的 token
  例如：swap(node_i.hop_2, node_j.hop_2)
  
效果：
  - 生成更多样化的 token 序列
  - 增强模型泛化能力
  - 利用全局语义信息

关键特点

特点说明
语义驱动基于节点语义相似性进行 swapping
数据增强Token swapping 作为一种数据增强
Center Alignment Loss约束多个 token 序列的表示学习

4. Contrastive Learning for Tokenized GT (NeurIPS 2024)

核心思想

使用对比学习增强 tokenized graph transformer 的节点表示。

关键贡献

  • 设计 tokenized GT 专用的对比学习目标
  • 增强 token 序列的表示能力
  • 提升节点分类性能

与 Hop-Offset 的关系

Offset 可设计专用的对比学习目标！
  • 正样本：同一节点不同邻居采样的 Offset
  • 负样本：不同节点的 Offset
  • 目标：增强 Offset 表示的一致性和可分性

📊 NAGphormer vs VoxGFormer 对比

核心差异

NAGphormer

Hop2Token 过程：

对于节点 v:
  Token_0 = AGG(特征(v))
  Token_1 = AGG(Hop_1(v)的所有邻居)
  Token_2 = AGG(Hop_2(v)的所有邻居)
  
每个 hop → 1 个 token
总 token 数 = K + 1

VoxGFormer (当前)

Hop Sampling 过程：

对于节点 v:
  Token_0 = 特征(v)
  Token_1 = [特征(邻居_1), 特征(邻居_2), ...]
  Token_2 = [特征(邻居_1), 特征(邻居_2), ...]
  
每个 hop → 多个 token (采样)
总 token 数 = 1 + K × samples_per_hop

详细对比表

维度 NAGphormer VoxGFormer
Token 来源 每个 hop 邻居聚合 每个 hop 邻居采样
Token 数量 K+1 个 (固定) 1 + K × S 个 (可变)
邻居信息保留 聚合后丢失多样性 采样保留多样性
差分信息 ❌ 无 ✅ Delta / Offset
Positional Encoding 可学习 PE Cosine PE
任务适配 通用节点分类 图异常检测 (GAD)

方向 A: Offset 方向语义分析

核心问题

Offset 向量的方向是否携带与异常检测相关的语义信息？

分析方法

1. 可视化分析：
   - 正常节点 vs 异常节点的 Offset 方向分布
   - 使用 t-SNE / PCA 降维可视化
   
2. 统计分析：
   - 计算 Offset 方向与异常标签的相关性
   - 分析不同数据集上的方向一致性
   
3. 理论分析：
   - Offset 方向是否指向"异常方向"？
   - 正常节点的 Offset 方向是否聚集？

预期发现

  • 如果 Offset 方向与异常相关 → 可以设计方向感知的异常分数
  • 如果正常节点 Offset 聚集 → 可以用方向偏离度作为异常指标

方向 B: Contrastive Offset Learning

核心思想

借鉴 Jinsong Chen NeurIPS 2024 论文，设计 Offset 专用的对比学习目标。

实现方式

def contrastive_offset_loss(node_i, node_j, temperature=0.2):
    """
    Offset 对比学习损失
    
    正样本：同一节点不同邻居采样的 Offset
    负样本：不同节点的 Offset
    """
    offset_i = compute_offset(node_i)  # [K, D]
    offset_i_aug = compute_offset(node_i, different_sampling)  # 增强版本
    offset_j = compute_offset(node_j)
    
    # 正样本相似度
    pos_sim = cosine_similarity(offset_i, offset_i_aug)
    
    # 负样本相似度
    neg_sim = cosine_similarity(offset_i, offset_j)
    
    # InfoNCE Loss
    loss = -log(exp(pos_sim/τ) / (exp(pos_sim/τ) + exp(neg_sim/τ)))
    return loss

预期效果

  • 增强 Offset 表示的一致性（同一节点不同采样）
  • 增强 Offset 表示的可分性（不同节点）
  • 提升异常检测性能

方向 C: Composite Offset Token

核心思想

借鉴 NTFormer 的 Node2Par，将 Offset 作为多种 token 类型之一。

Token 类型设计

tokens = {
    'original': hop_features,           # 原始 hop 特征
    'offset': hop_k - hop_0,           # Hop-Offset
    'delta': hop_k - hop_{k-1},        # Delta
    'degree': node_degree,              # 度特征
    'structure': pagerank, betweenness, # 结构特征
}

# 输入 Transformer
input_sequence = concat(
    tokens['offset'],
    tokens['structure'],
    tokens['degree']
)

预期效果

  • 多视角表示，更全面的节点特征
  • Offset 与结构特征互补
  • 无需针对不同图类型修改模型

方向 D: Offset Token Swapping

核心思想

借鉴 SwapGT，利用 Offset 方向的语义相关性进行 token swapping。

实现方式

def offset_based_swapping(node_i, node_j, threshold=0.8):
    """
    基于 Offset 相似性的 Token Swapping
    """
    offset_i = compute_offset(node_i)
    offset_j = compute_offset(node_j)
    
    # 计算 Offset 方向相似度
    similarity = cosine_similarity(offset_i, offset_j)
    
    if similarity > threshold:
        # Offset 方向相似 → 可以交换某些 hop token
        # 增强表示多样性
        swap_token(node_i, node_j, hop_k)

预期效果

  • 增强 Offset 表示的多样性
  • 利用全局语义信息
  • 提升模型泛化能力

🎯 结论与建议

核心发现

1. Hop-Offset 是 VoxGFormer 独特的差分表示

Jinsong Chen 的 NAGphormer、NTFormer、SwapGT 都没有使用差分信息。Hop-Offset 是 VoxGFormer 的独特优势。

2. 当前对 Hop-Offset 的挖掘还不够深入

特别是以下方面：

  • Offset 方向语义：方向是否指向"异常"？
  • 对比学习增强：增强 Offset 表示的一致性
  • 多 token 类型组合：Offset 与其他特征的互补

3. Jinsong Chen 的工作提供了很好的参考

  • SwapGT 的 token swapping 思想 → 可用于 Offset
  • NTFormer 的多 token 类型组合 → Offset 作为其中一种
  • Contrastive Learning 的表示增强 → Offset 对比目标

注意事项

  • 不要简单照搬：Jinsong Chen 的工作针对节点分类，需要适配异常检测任务
  • 保持 Hop-Offset 的独特性：差分信息是优势，不要丢失
  • 先分析后改进：P1 的 Offset 方向语义分析是基础
