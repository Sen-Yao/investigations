## 二、深度分析

### 2.1 最佳向量聚合方法

| 数据集 | 最佳方法 | 向量AUC |
|--------|----------|---------|
| t_finance | flatten | 0.9464 |
| Amazon | flatten | 0.8689 |
| tolokers | flatten | 0.7070 |
| photo | flatten | 1.0000 |
| elliptic | **max** | 0.7598 |
| reddit | flatten | 0.7705 |

**结论**: `flatten`方法在5/6数据集上最优，`max`方法在elliptic数据集上最优。

### 2.2 信息冗余度分析

| 数据集 | 平均相关性 | 冗余程度 |
|--------|-----------|---------|
| t_finance | 0.9092 | 中等 |
| Amazon | 0.9251 | 高 |
| elliptic | 0.9739 | 高 |
| photo | 0.9922 | 极高 |
| tolokers | 0.9944 | 极高 |
| reddit | 0.9978 | 极高 |

**发现**: Delta向量与原始Token相关性普遍较高（>0.9），但仍能提供显著的异常检测增益。

### 2.3 与Delta范数对比

Delta向量比Delta范数更有效的原因：
1. **信息完整性**：保留每个特征维度的变化方向和幅度
2. **可学习权重**：允许模型学习各特征维度的重要性
3. **区分能力**：不同特征维度对异常的贡献不同

## 三、对PromptGAD的改进建议

### 方案1：Delta向量作为辅助Token序列
```python
# 原始Token: [N, K+1, D]
# Delta向量: [N, K, D]
delta_tokens = node_tokens[:, 1:] - node_tokens[:, :-1]
extended_tokens = torch.cat([node_tokens, delta_tokens], dim=1)  # [N, 2K+1, D]
```

### 方案2：残差增强Token
```python
enhanced_tokens = node_tokens[:, :-1] + alpha * delta_tokens  # [N, K, D]
```

### 方案3：特征加权Delta
根据各数据集的特征重要性差异，学习特征维度权重。

## 四、结论

**Delta向量（不取L2范数）在所有6个数据集上都显著优于Delta范数，平均提升+0.17 AUC。**

- 最佳场景（photo）：AUC从0.61提升至1.00
- 最差场景（elliptic）：仍有+0.09的提升
- 推荐方法：`flatten`聚合方式

**但需注意**：Delta向量与原始Token存在较高信息冗余（相关性>0.9），应作为**辅助特征**而非替代原始Token。
