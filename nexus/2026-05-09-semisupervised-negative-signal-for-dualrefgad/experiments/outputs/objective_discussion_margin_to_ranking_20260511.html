DualRefGAD 可学习 head 的优化目标：从 margin regression 转向半监督 anomaly ranking

Nexus ｜ 理论讨论与路线建议 ｜ 2026-05-11

摘要：当前 probe 实验表明，以 margin regression 为目标的 MLP 可以在整体排序上高度拟合 closed-form directional margin，但仍无法完全保留异常检测所需的 tail ranking 能力。这说明 margin regression 作为一个数值拟合任务，并不天然等同于 anomaly ranking。更优雅的目标应该直接对齐半监督异常检测的本质：在只有正常标签的情况下，让模型学会把偏离正常分布的节点排序更高。本邮件讨论三类候选优化目标：normal-only pairwise ranking with pseudo-anomaly candidates、contrastive normality readout、以及基于 pseudo-labels 的 ranking surrogate loss，并给出推荐路线。

一、现状：margin regression 不是最终 anomaly detection objective

上一轮 probe 的训练目标是在 normal-only nodes 上用 MSE 拟合 closed-form margin cos(u,d)。从 MSE 和 Spearman 角度看，normalized input 的 MLP 已经成功学会整体 margin 分布；但从 AUC/AP/top-k 角度看，best head 的异常检测效用仍略低于 margin-only baseline，尤其是 AP 和 top1% anomaly ratio。

Insight：MLP 可以学会整体排序结构，但在最关键的异常尾部上会压低 score，导致 top-k 检测能力下降。这说明 margin regression 本质上是一个数值拟合任务，而 anomaly detection 是一个 ranking 任务；二者在高相关性下仍可能在 tail 上不一致。

二、半监督异常检测的真正目标是什么

在只有正常标签 L_n 的情况下，异常检测的目标不能写成经典 supervised binary classification。更合理的表述是：

半监督异常检测的本质目标

也就是说，我们希望模型输出一个 score s(v)，使得：

正常节点应比偏离节点得分更低

这里 v_{deviating} 不是已知异常标签，而是由正常分布模型诱导出的“偏离候选”。这正是 DualRefGAD 中 anomaly-side references 的角色：它们不是 ground-truth anomalies，而是由 normal rejection 与 local similarity 共同构造的 pseudo-anomaly candidates。

三、候选优化目标：从 margin regression 转向 anomaly ranking

方案 A：Normal-only pairwise ranking with pseudo-anomaly candidates

在已有正常分布模型（如 diag-Gaussian / PCA residual）的前提下，可以为每个正常节点 v_i \in L_n 构造一组 pseudo-anomaly candidates C_i。这些候选来自 normal rejection score 较高的节点，例如 g_a(v) > threshold。然后训练 pairwise ranking loss：

pairwise ranking: pseudo-anomaly 应比正常得分高

这个目标直接对齐 anomaly ranking：它不要求 head 输出精确 margin 数值，只要求 pseudo-anomaly candidates 排在正常节点之上。风险在于 pseudo-anomaly candidates 可能包含误判，但这是半监督设定下不可避免的噪声。

方案 B：Contrastive normality readout

不显式构造 pseudo-anomaly candidates，而是用 contrastive loss 把正常节点的 relation vectors u_i = h_i - r_n(i) 拉向 normal reference center，同时把偏离候选的 u_j 推开：

contrastive normality: 正常 relation 应接近 normal center

这个方案的直觉是：正常节点不仅在属性上接近正常参考，而且在 relation geometry 上也应接近 normal center；偏离节点则在 relation space 上远离。它更强调几何一致性，而不是 score ranking。

方案 C：Pseudo-label ranking surrogate（AUC/AP proxy）

如果接受 pseudo-labels 作为 soft anomaly 标签，可以直接训练 ranking surrogate loss（如 listwise ranking 或 AUC proxy）。pseudo-labels 可以来自 normal rejection prior：

y_i^{\\text{pseudo}} = g_a(v_i)

pseudo-label ranking surrogate

这个方案风险更大，因为 pseudo-labels 不可避免有噪声；但如果 rejection prior 较可靠，它可以直接对齐 AUC/AP 目标。

四、推荐路线：先做方案 A，再对比方案 B

我推荐先测试方案 A：normal-only pairwise ranking with pseudo-anomaly candidates。理由有三：

  1. 与当前 DualRef 生态一致：pseudo-anomaly candidates 可以直接使用现有 R_a(v) 或其子集，不需要重新设计 rejection mechanism。
  2. 目标对齐 anomaly ranking：ranking loss 直接训练“pseudo-anomaly 排在正常之上”，而不是拟合 margin 数值。
  3. 可控风险：虽然 pseudo-anomaly candidates 有噪声，但 pairwise ranking 只要求相对排序，对个别误判样本有一定容忍度。

方案 B 作为后续备选：contrastive normality readout 更强调几何一致性，可能更稳定，但它不显式训练 ranking，需要额外评估是否对齐 top-k anomaly detection。

Recommendation：下一步实验应该从 margin regression 转向 pairwise ranking with pseudo-anomaly candidates。输入仍使用 ud_prod_absdiff_norm，训练目标改为 pairwise ranking：normal nodes vs pseudo-anomaly candidates from R_a。关键检查是：ranking loss 是否在保持 AUC/AP 的同时，改善或至少不破坏 top-k anomaly ratio。如果 ranking loss 再次破坏 tail ranking，则需要进一步诊断 pseudo-anomaly candidates 的 purity 和 pairwise 构造规则。

五、需要补齐的理论支撑

无论选择哪个方案，都需要补齐以下理论支撑：

  • Pseudo-anomaly candidates 的定义与 purity：它们来自 normal rejection + local similarity，但不是 ground-truth anomaly。需要量化其中真实异常的比例（已在 margin-only 诊断中得到 anomaly-side reference purity ≈0.1074，对真实异常节点的 anomaly-side reference anomaly ratio ≈0.6837）。
  • Pairwise 构造的合法性：pairwise ranking 不使用真实 anomaly labels，因此符合半监督约束；但需要说明 pseudo-anomaly candidates 是由正常分布模型诱导出的偏离候选，而不是手工注入的异常标签。
  • 与 closed-form margin 的关系：ranking loss 不显式使用 margin 数值，但 head 的输入仍基于 relation vectors u 与 d。理论上应该说明：ranking loss 是在 relation geometry 上学习一个 monotonic transformation，而不是替换整个 reference-conditioned coordinate system。

六、Next Actions

  1. 构造 pairwise ranking probe：输入 ud_prod_absdiff_norm，训练目标改为 pairwise ranking：normal nodes vs pseudo-anomaly candidates from R_a。监控 AUC/AP/top-k，并与 margin-only baseline 对比。
  2. 如果 ranking loss 破坏 tail ranking：诊断 pairwise 构造规则（例如只用 high-purity pseudo-anomaly candidates），或尝试 contrastive normality readout。
  3. 如果 ranking loss 成功：扩展到 seeds [0,1,2,3,4]，确认稳定性，然后考虑是否将其作为正式可学习 head 方案。

当前最重要判断是：margin regression 作为诊断实验已经完成，下一步应转向直接对齐 anomaly ranking 的训练目标。
