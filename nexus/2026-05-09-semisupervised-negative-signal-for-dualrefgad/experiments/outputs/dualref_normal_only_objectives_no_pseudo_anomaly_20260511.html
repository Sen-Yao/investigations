DualRefGAD normal-only 训练目标：不依赖伪异常生成的几个数据驱动方案

发送者：Nexus ｜ 场景：可学习 head / semi-supervised normal-only objective ｜ 日期：2026-05-11

摘要：我同意你对 pseudo-anomaly shell 的批评。若训练阶段没有见过异常，显式生成伪异常会把问题偷换成“研究者定义异常”。更优雅的主线应是 normal-only relation distribution modeling：只学习 labeled normal nodes 在 DualRef relation space 中的规律，推理时用低 likelihood、高 energy 或低 consistency 作为异常证据。首推方案是 Conditional Normalizing Flow，其次是 reference-view consistency，再其次是 score matching / one-class relation objective。

一、问题重定义：从“生成异常”转为“学习正常”

上一版 shell ranking 的问题不只是超参数多，而是语义上过强。γ、R_min、R_max 规定了伪异常应该沿哪个方向、在多远处出现。这在工程上可以构造 hard negative，但在 normal-only 设定下会带来逻辑负担：我们并没有观察到异常分布，因此不能自然声称这些生成样本代表异常。

更稳妥的机器学习表述是 one-class / density / energy modeling：我们只声称“知道正常关系大致如何分布”，不声称“知道异常如何生成”。这使训练目标与监督信息严格匹配。异常分数来自对正常分布的偏离，而不是来自人为构造的负类。

核心判断：DualRefGAD 的可学习 head 应首先被定义为 normal relation distribution estimator，而不是 normal-vs-pseudo-anomaly classifier。

二、共同建模对象：DualRef relation feature

所有候选方案都应保留 DualRef 的解释性变量，而不是退回到普通 embedding classifier。

DualRef relation variable

其中 u_i 是节点相对 normal reference 的偏移，d_i 是 anomaly-side reference 与 normal reference 之间的方向，c_i 是可选条件上下文。第一版 diagnostic 可以沿用之前证明有效的 normalized interaction feature：


u = h - r_n
d = r_a - r_n
phi = concat(
    normalize(u), normalize(d),
    normalize(u) * normalize(d),
    abs(normalize(u) - normalize(d)),
    context_features
)
train_nodes = labeled_normal_nodes_only
loss = -log_prob(phi[train_nodes] | context[train_nodes])
score = -log_prob(phi | context)   # labels only used for diagnostics

Insight：这样做把 closed-form margin 中的物理结构保留下来，同时把评分函数从手写 cosine 推向数据驱动学习。

三、方案 A：Conditional Normalizing Flow（首推）

Flow 方案直接学习 labeled normal relation 的条件概率密度：

normal-only conditional likelihood

使用可逆变换时，log-likelihood 可由 base density 与 Jacobian determinant 精确给出：

conditional normalizing flow

这个目标没有定义任何伪异常。它只要求正常节点的 relation features 在模型下有高概率；测试时低概率样本自然得到高 anomaly score。

维度优点风险诊断方式
理论normal-only likelihood，与监督信息一致高维 likelihood 可能受 typicality / scale 影响记录 NLL 与 feature norm、degree、reference distance 的相关性
工程不需要伪异常，不需要 shell 半径需要选择 flow 容量先用 small RealNVP/MAF，normal validation NLL early stop
解释score 是 relation feature 的 negative log-likelihood单纯 NLL 可能不如 margin 直观报告 per-feature contribution / ablation

Insight：Flow 是最接近“数据驱动 normality modeling”的方案。它的超参数属于标准模型容量选择，而不是人为规定异常边界。

四、方案 B：Reference-view Consistency（VICReg / Barlow 风格）

这个方案不显式估计密度，而是构造同一个 labeled normal node 的两个 reference views。例如对 normal reference set 进行 bootstrap，得到 r_n^{(a)}(i) 与 r_n^{(b)}(i)，然后要求同一正常节点的 relation representation 在两种 reference views 下保持一致。

reference-view consistency

这里第一项是 invariance，要求同一节点不同 reference view 的表示接近；variance 项防止 collapse；covariance 项减少维度冗余。它比 InfoNCE 更适合当前问题，因为它不需要把 batch 内其他 normal 节点当作 negative，也不需要伪异常。

性质说明
不使用负样本避免“谁是 negative”的哲学问题
利用 DualRef 随机性reference set 的 bootstrap 提供 natural views
异常分数测试时可用 cross-view inconsistency 或 projection-space deviation

Insight：Consistency 方案更像 RHO/MAG 的 alignment 思想，但它被放回 DualRef relation space 中，不再是 global center 或通用图增强。

五、方案 C：Energy / Denoising Score Matching

Energy-based normality model 也不需要伪异常。它学习 normal relation distribution 的 score function，即 log-density 对输入的梯度。用 denoising score matching 可以只在 normal samples 附近训练：

denoising score matching

这个方案理论上优雅，因为模型学习的是“如何从 normal manifold 附近的扰动回到高密度区域”。但它比 flow 更难诊断，异常分数也不如 NLL 直接。若作为第一阶段实验，我不建议优先做；它更适合在 flow/consistency 有积极信号后，作为更强表达能力的后续方案。

六、方案 D：Relation-space Deep SVDD / One-class

最简单的 normal-only baseline 是把 relation features 映射到一个 latent space，并压向 normal center：

relation-space one-class objective

它很干净、易实现，但我不建议把它作为最终方案，因为 global center 假设可能削弱 DualRef 的 local reference-conditioned geometry。这一点和 RHO 类似：one-class center 是可借鉴的训练思想，但不能让它覆盖 DualRef 的局部关系优势。

七、是否保留 closed-form margin

我建议在第一阶段保留 closed-form margin 作为 conservative baseline，而不是马上删除。一个安全做法是只在诊断中比较 pure density score 与 hybrid score：

optional conservative hybrid score

这里 α 不是最终方法必须引入的核心超参数，而是诊断用：它帮助判断 learned density 是否提供了 margin 之外的额外信息。如果 pure NLL 已经优于 margin，就不需要 hybrid；如果 hybrid 最好，说明 density model 捕获了互补信号，但尚不足以完全替代几何 margin。

八、推荐实验路线

建议：第一步做 relation_density_probe，不是 relation_shell_ranking_probe。训练目标使用 Conditional Flow 的 normal-only NLL；训练节点只包含 labeled normal；评价使用 AUC/AP/top1/top5/tail error，仅作为 test-side diagnostics。
优先级实验成功标准
1Flow NLL on ud_prod_absdiff_normAUC/AP 接近或超过 margin-only，并改善 tail ranking
2Flow + context conditioning排除 degree/reference-scale confound 后仍有效
3Reference-view VICReg consistencyconsistency error 与 anomaly label 有稳定相关性
4Score matching / energy作为高表达后续，不作为首轮

Insight：这条路线把研究问题从“如何制造伪异常”转成“正常 relation 的分布是否足以支撑 anomaly ranking”。这更符合 normal-only semi-supervised GAD 的信息结构，也更容易写成一套清晰的方法论。
