DualRefGAD 阶段诊断：Relation MLP 无法恢复 margin，问题不只是 pairwise objective

Nexus ｜ 2026-05-09 ｜ Elliptic / seed 0 / experiment-runner skill

摘要：本次邮件汇总两个连续 probe：先前的 high-dimensional Relation MLP + normal-only pairwise ranking 已明显失败；随后我们做了 margin-regression 诊断，直接让 MLP 从 relation vector 回归 closed-form margin，但最好的异常检测结果仍只有 AUC 0.5732 / AP 0.1103，远低于 margin-only 的 AUC 0.7938 / AP 0.5510。因此，当前结论要进一步收紧：失败不只是 pairwise mismatch objective 不对齐，MLP 对高维 relation 输入也没有自然恢复出 cosine margin 的有效排序。下一步不应继续盲目调普通 MLP，而应显式研究 cosine / bilinear / normalized relation 这类几何归纳偏置。

一、背景：为什么要做 margin-regression 诊断

前一个 probe 的目标是验证一个看起来合理的假设：既然 closed-form margin 表现很好，那么构成它的输入 u = h_v - r_n 与 d = r_a - r_n 理论上应当包含足够信息；如果把这些 relation vector 交给一个 MLP，再用 normal-only mismatch pairwise ranking 训练，模型应该至少能接近 margin。

但实际结果不支持这个假设。Relation MLP + pairwise ranking 的最佳 AP 只有 0.1119，而 margin-only AP 是 0.5510。这个差距太大，不能归因于普通超参数波动。因此我们拆出一个更基础的诊断问题：如果不再使用 pairwise mismatch objective，而是直接让 MLP 回归 margin，模型能否恢复 margin 的异常排序？

这个诊断不使用真实 anomaly labels 作为训练目标；它只把 closed-form margin 当作一个由当前 frozen embedding 和 frozen references 计算出的几何量。若 MLP 能够回归 margin 并恢复 AP，那么前一阶段失败主要是 objective 不对齐。若 MLP 仍然不能恢复 margin，那么问题就更深：普通 MLP 在当前输入、归一化和训练设置下，并不能自然重建 cosine margin 这个几何归纳偏置。

三、主要发现

第一，失败原因不能再简单归结为 pairwise objective 不对齐。pairwise 实验失败时，我们的解释是 normal-only mismatch target 可能没有对应真实异常排序。这个解释仍然成立，但 margin-regression 结果说明它不是唯一瓶颈：即使直接给 MLP 一个更明确的 margin target，最终 score 的异常排序仍然接近随机到弱信号区间，而不是接近 margin baseline。

第二，closed-form margin 的强度更像来自几何归纳偏置，而不只是来自输入信息量。margin 是对两个 relation direction 的归一化角度比较，它强制消除了 norm 尺度，并把排序集中在方向一致性上。普通 MLP 虽然有表达能力，但它面对的是未显式归一化的高维向量；在 normal-only target 分布上学习 MSE 时，它并没有被强制学到“先 normalize，再做 direction similarity”的结构。

第三，加入更多原始信息不一定有用。h_rn_ra_ud 并没有超过 ud_prod_absdiff，说明把 h、r_n、r_a 全部塞进 MLP 不会自动带来更好排序，反而可能增加自由度与无关变化。当前最有效的输入仍是围绕 u、d 及其简单交互项的 relation representation。

Key Finding：当前最重要的判断是：普通 MLP head 不是 margin 的自然替代品。margin 的有效性来自明确的 normalized directional similarity。若要做可学习版本，应该学习的是“可参数化的方向相似度 / 双线性角度度量 / normalized relation metric”，而不是继续把原始 relation vector 交给普通 MLP 期待它自行发现 cosine。

四、对研究路线的影响

这组结果会改变我们对“可学习分类头”的定义。原先的想法是：margin 输入里有信息，所以 MLP 分类头应该能学到并超过 margin。实验现在说明，这个说法需要加一个条件：head 的结构必须保留或显式表达 margin 的几何归纳偏置。如果 head 完全自由，它并不会自动朝这个方向收敛；如果训练信号又只来自 normal-only proxy，这种偏离会更明显。

因此，后续不建议继续做普通 MLP 的 hidden、dropout、lr 大规模 sweep。当前最好 AP 只有 0.1103，距离 0.5510 太远。继续扩大普通 MLP 的容量，更可能得到的是更会拟合 normal margin 分布的模型，而不是更好的异常排序器。

更合理的方向是把“可学习”放在 metric 层，而不是完全自由的 classifier 层。例如可以学习一个对 u 和 d 的投影，然后计算归一化相似度；也可以学习一个低秩双线性矩阵来替代普通 dot product；还可以学习温度、维度权重或 feature-wise gate，但最终仍保持 normalized directional score 的结构。这类方法保留深度学习可学习性，同时不放弃 margin 已经证明有效的几何核心。

五、建议的下一步实验

我建议下一步做 Parameterized Directional Metric Head，而不是普通 MLP。形式可以从最小版本开始：

u = h_v - r_n
d = r_a - r_n
u' = W_u u
d' = W_d d
score = cos(u', d')

这个 head 仍然是可学习的，但它把学习限制在“如何重加权和投影 direction space”上，而不是让模型任意生成 score。它比 residual 更像一个真正的方法模块，也比普通 MLP 更尊重实验已经验证的结构事实。

训练策略上，我建议先做两类诊断。第一类是 margin-regression：看 parameterized metric 能否恢复 margin 排序；第二类是 normal-only pairwise：看它是否比普通 MLP 更稳定。如果 metric head 在 margin-regression 中能接近 0.55 AP，而普通 MLP 不能，就可以明确写出一个方法洞察：DualRef anomaly signal is metric-structured rather than classifier-structured。

Recommendation：停止普通 MLP head 的超参数扩展，转向 normalized / bilinear / projected cosine 形式的可学习方向度量。这个建议的取舍是：我们牺牲一部分自由函数表达能力，但换来与已验证 margin 信号一致的结构约束；这比 residual 更自然，也比自由 MLP 更可控。

六、Next Actions

如果你同意，我建议下一步启动一个小型 probe：stage1_parameterized_directional_metric_seed0。它只比较三种 head：identity cosine baseline、diagonal weighted cosine、low-rank projected cosine。训练目标先用 margin-regression 做表达能力验证，再用 normal-only pairwise 做目标对齐验证。这个实验能直接回答：是否存在一个“有深度学习味道但不丢失 margin 几何核心”的 head。
