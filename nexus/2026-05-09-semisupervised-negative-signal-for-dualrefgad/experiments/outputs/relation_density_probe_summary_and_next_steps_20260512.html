Relation Density Probe 结果总结与下一步：normal-only density 不足以替代 DualRef margin

发送者：Nexus ｜ 场景：DualRefGAD 可学习 head / semi-supervised normal-only objective ｜ 日期：2026-05-12

摘要：昨晚完成的 relation_density_probe 实验给出了一个重要但偏负面的信号：单纯学习 labeled normal nodes 的 relation feature 分布密度，并用 negative log-likelihood 作为异常分数，其 AUC 远低于 DualRef closed-form margin baseline。这说明 normal-only density modeling 不是我们问题的正确表述；DualRef 的 anomaly reference direction d=r_a-r_n 不能被简单消解成一个 latent variable，而必须以更直接的 geometry-aware 形式进入 scoring function。下一步建议尝试 likelihood-ratio with anomaly direction conditioning 或 margin + learned correction，而不是继续堆更复杂的 flow。

一、实验设置回顾

这次实验延续了上次邮件提出的 normal-only density modeling 思路，试图回答一个核心问题：在只有 labeled normal nodes 的 semi-supervised GAD 设定下，是否可以通过学习 normal relation distribution p(φ) 来替代 closed-form margin m=cos(u,d)。

实验对象是 DualRef relation feature φ_i，沿用之前 MSE probe 中证明有效的 ud_prod_absdiff_norm 输入模式，即：


phi_i = concat(
    normalize(u_i),
    normalize(d_i),
    normalize(u_i) * normalize(d_i),
    |normalize(u_i) - normalize(d_i)|
)
u_i = h_i - r_n(i)
d_i = r_a(i) - r_n(i)

训练目标只用 labeled normal nodes 的 negative log-likelihood：

s_i = -log p_θ(φ_i | c_i), φ 是 relation feature

推理时异常分数为 -log p(φ)。异常标签只在 test set diagnostics 中使用，不进入训练 loss。

我们在 elliptic 数据集上跑了 8 个配置，包括 diag Gaussian 和 RealNVP flow，hidden/layer 维度分别为 128/256 和 4/8：

Insight：无论是简单 diag Gaussian 还是更复杂的 RealNVP flow，其 AUC 都远低于同一 match-ref 构造下的 closed-form margin baseline（约 0.7938 AUC，0.5510 AP）。RealNVP 反而比 Gaussian 更差，说明训练更复杂 density model 并没有帮助，甚至可能因为 normal-only 样本量小、高维 relation feature 空间难以准确估计而引入噪声。

三、为什么 density approach 失败？

这个结果其实不意外，甚至有点启发。我之前邮件里乐观地认为“只要学习 normal relation distribution，偏离该分布的节点自然得到高异常分数”。但实验表明，这个表述对 DualRef 的核心变量 d=r_a-r_n 不够尊重：

m_i = cos(u_i, d_i), where u_i = h_i - r_n(i), d_i = r_a(i) - r_n(i)

问题在于：d 本身是由 anomaly-side reference 构造的方向，它已经携带了“偏离正常应该朝哪里”的信息。如果我们把 d 消解成一个 latent dimension，让它像其他特征一样被 density model 吸收，那我们就丢弃了 DualRef 的一个关键先验：异常的方向不是任意的，而是由 anomaly reference set 预先指向的。

换句话说，closed-form margin m=cos(u,d) 的成功在于它显式利用 anomaly direction d；而 density approach 的失败在于它把 d 当成一个普通 latent variable，试图从 normal samples 中推断“异常应该是什么方向”，但 normal-only training 没有这个信息。

核心判断：DualRef 的 anomaly reference direction d 不能被简单地 latent-space density modeling 消解；它必须以 conditioning 或 explicit geometry 的形式进入 scoring function。Normal-only density modeling 的失败说明，我们之前把“学习异常检测边界”表述为“学习正常分布密度”是不够精确的；更精确的表述应该是“学习正常分布，但异常分数来自偏离正常分布沿 anomaly direction的程度”。

四、修正后的研究表述

基于这次负面结果，我建议把问题表述从“normal-only density modeling”修正为更 geometry-aware 的形式：

  1. 不放弃 anomaly direction d：density 可以学，但 anomaly score 不应该只是 -log p(φ)，而应该和 d 有更直接的交互。
  2. likelihood-ratio with anomaly direction conditioning：训练一个 conditional density p(φ | context)，其中 context 可以是 anomaly direction 相关变量，然后异常分数来自 likelihood ratio：
s_i = log p(φ_i | anom-context) - log p(φ_i | normal-context)

这个思路不要求显式生成伪异常，而是用 d 作为 conditioning variable，让模型学会“沿着 d 方向的 φ 更应该被判定为异常”。训练时仍然只用 labeled normal nodes，但 conditioning 信息让模型隐式感知 anomaly direction。

另一种路线：margin + learned correction

如果不想引入复杂的 conditional density，也可以采用更保守的设计：保留 closed-form margin 作为主 score，让 learnable head 只做 calibration：

s_i = m_i + β·f_θ(φ_i) or s_i = α·[-log p(φ_i)] + (1-α)·m_i

这样 head 的任务不再是“替代 margin”，而是“修正 margin 的局部偏差”。这和之前 MSE probe 不同：MSE probe 试图让 MLP 直接拟合 margin，结果是学得很好（Spearman≈0.99）但 tail ranking 有损失；而 calibration 模式的目标是保住 margin 的全局排序，同时修正局部 tail 错误。

五、不建议继续的方向

基于这次实验，我不建议：

  • 继续堆更复杂的 flow（如 NSF、MAF、神经 spline）：这次 RealNVP 已经比 Gaussian 更差，说明问题不是模型表达能力，而是 objective 本身和 anomaly detection 的 tail ranking 不对齐。
  • 在高维 relation feature 上做更精细的 density estimation：normal-only 样本量有限，高维密度估计本质困难；而且即便估计准确，NLL 和 anomaly ranking 的关系也不是我们想要的。
  • 把 anomaly direction d 完全消解进 latent variable：这丢弃了 DualRef 的核心先验。

六、建议的下一步实验

建议：下一步不应该继续做“纯 density”，而是转向 geometry-aware normality modeling。最优先的候选是 likelihood-ratio with anomaly direction conditioning，其次是 margin + learned calibration。

Insight：这次 density probe 的负面结果其实帮助收敛了研究问题。它说明我们之前“从 margin regression 转向 normal-only density”的转向不够彻底，因为 density 仍然没有充分利用 DualRef 的 anomaly direction 先验。下一步应该做的是把 anomaly direction 显式引入 normality model，而不是试图从 normal samples 中隐式推断它。

下一步：如果你认可 likelihood-ratio / margin-calibration 方向，我建议先做一个 seed0 diagnostic：训练 conditional density p(φ | d)，异常分数为 likelihood ratio p(φ | normal-context) / p(φ | anom-context)。我会先写一个轻量脚本，参考之前 density probe 的架构，但把 conditioning variable 设计为与 d 相关的统计量（如 ||d||、d_norm 或 anomaly-side rejection score）。正式实验前会按 experiment-runner / experiment-audit 规范再请求你确认。
