DualRefGAD Tokenization 完整说明：从节点、descriptor 到 normal / anomaly-side references

Nexus ｜ 2026-05-11 ｜ 面向不了解方法读者的学术版解释

摘要：本邮件重新从零定义当前 AUC≈0.79 方案中的 tokenization。所谓 DualRef tokenization，不只是把若干邻居拼成序列，而是为每个目标节点构造一个局部的 reference-conditioned coordinate system：正常参考 normal references 给出局部正常原点，异常侧参考 anomaly-side references 给出可疑偏移方向。当前有效配置为 descriptor_mode=hop_attr、gn_mode=label_gate、ln_mode=descriptor_similarity、ga_mode=normal_soft_or、la_mode=descriptor_similarity，最终 token 序列为 [target; 4 normal refs; 16 anomaly-side refs]，并用 cos(h_v-r_n, r_a-r_n) 作为异常分数。

一、问题设定：半监督图异常检测中的可用信息

设图为 G=(V,E)，节点数为 n。每个节点 v_i 具有原始属性向量 x_i，所有节点属性组成矩阵 X。半监督异常检测设置下，我们只有一个已知正常训练集合 L_n。这个集合必须只包含正常节点，即训练前需要满足 sum(labels[train_idx]) == 0。真实异常标签不允许进入训练过程，也不允许直接用于 reference selection；它只能用于最终评估 AUC/AP 或诊断 reference purity。

半监督正常训练集合

在这个约束下，方法不能直接学习一个 normal-vs-anomaly 分类器。DualRefGAD 的思路是：虽然没有异常监督，我们可以从正常分布的偏离程度中构造一个 anomaly-side prior，并与局部 descriptor similarity 结合，为每个目标节点生成一组“异常侧参考”。这组参考不是 ground-truth anomaly，而是由正常模型诱导出的 suspicious reference candidates。

二、Descriptor Construction：descriptor_mode=hop_attr 是什么

Reference selection 不直接在 Transformer embedding 上做，因为 tokenization 发生在 encoder 之前。我们首先为每个节点构造 descriptor z_i，用它衡量节点之间的结构-属性相似性。当前 AUC≈0.79 的配置使用 descriptor_mode=hop_attr，即把节点的原始属性与多跳邻域聚合属性拼接起来，形成局部结构感知的描述符。

直观地说，hop_attr 不只看节点自己的属性，还看它在图中若干跳范围内的属性环境。若用 A 表示邻接矩阵、D 表示度矩阵，P=D^-1A 表示随机游走式邻域聚合算子，则 2-hop hop_attr 可以写成如下形式：

hop_attr descriptor，hops=2

这里的 hops=2 表示 descriptor 包含 0-hop、1-hop、2-hop 的属性信息。代码中还有 rwse 与 hop_attr_rwse 两种 descriptor，但当前 0.79 方案明确使用 hop_attr。这是重要细节，因为此前较低性能的配置混入过 hop_attr_rwse 和不同 reference gate，不能与当前方案混淆。

三、Normal Model：如何从正常节点估计偏离程度

为了生成 anomaly-side prior，方法需要一个“正常分布模型”。当前配置使用 pn_estimator=pca_residual。具体做法是：只用已知正常集合 L_n 的 descriptor 拟合一个 PCA 子空间，然后把所有节点投影到这个正常子空间并重构。若节点的 descriptor 难以被正常子空间重构，则它在 normal model 下具有更高 residual，意味着它更可能是异常侧候选。

仅用正常节点拟合 PCA normal subspace

PCA residual 作为偏离正常子空间的信号

需要强调的是，PCA normal model 不是最终异常检测器。它只是 reference generation 的一部分，用于构造 anomaly-side prior。最终 AUC≈0.79 的 score 并不是 residual norm，而是后续 DualRef embedding 上的 directional margin。

四、Normal Reference Generation：正常参考如何生成

对每个目标节点 v_i，normal references 的作用是定义“相对于谁算正常”。当前配置为 gn_mode=label_gate 与 ln_mode=descriptor_similarity。这意味着 normal reference 候选池被严格限制为已知正常训练集合 L_n。方法先计算目标 descriptor z_i 与每个正常候选 z_j 的 cosine similarity，然后选择最相似的 normal_k=4 个正常节点作为 R_n(i)。

目标节点与正常候选之间的 descriptor similarity

normal references 的选择规则

这一步的学术意义是局部化 normality。与其用所有正常节点的全局中心作为唯一 normal prototype，不如为每个目标节点选择 descriptor 空间中最相似的几个已知正常节点，让 normal reference 更贴近目标节点的属性-结构语境。因此，后续偏移向量 h_i-r_n(i) 不是相对于全局正常中心，而是相对于目标节点自己的局部正常基准。

五、Anomaly-side Reference Generation：异常侧参考如何在无异常标签下生成

异常侧参考是最容易误解的部分。这里的 anomaly-side references 不是“真实异常标签挑出的异常节点”。当前配置完全不使用真实 anomaly labels 来生成它们。它们是由两个信号共同决定的：一是节点相对正常分布的全局偏离程度，二是与目标节点的局部 descriptor similarity。

当前配置 ga_mode=normal_soft_or 表示全局 anomaly-side prior 由两个 normality rejection 信号 soft-or 组合而来。代码中分别在 hop-attribute descriptor 与 RWSE descriptor 上拟合 diag-Gaussian normal model，得到两个 rank percentile rejection score，记作 a_i^hop 与 a_i^rwse。soft-or 组合为：

normal_soft_or anomaly-side prior

这个公式的含义是：只要一个节点在属性邻域统计或结构随机游走统计中明显偏离正常分布，它就会获得较高 anomaly-side prior。soft-or 不是强制两个信号都高，而是允许任一模态出现异常倾向。

随后，la_mode=descriptor_similarity 会计算目标节点 z_i 与任意候选节点 z_j 的 descriptor cosine similarity。最终 anomaly-side reference score 是局部相似性加全局 anomaly-side prior：

s_a(i,j)=\cos(z_i,z_j)+g_a(j),\quad j\ne i
R_a(i)=\operatorname{TopK}_{j\in V\setminus\{i\}}\ s_a(i,j),\quad K_a=16

这一步有一个重要设计直觉：如果只选全局最异常的节点，它们可能与目标节点语境无关；如果只选最相似节点，它们可能只是正常邻近样本。把 descriptor similarity 与 anomaly-side prior 相加，等价于寻找“既与目标节点处于相近语境、又更像正常分布外侧”的候选节点。这些节点构成 R_a(i)，用于定义目标节点的异常侧方向。

六、DualRef Tokenization：最终送入 Transformer 的序列

有了 R_n(i) 与 R_a(i) 后，tokenization 才真正发生。对于每个目标节点 v_i，方法构造一个长度为 1+K_n+K_a 的 token sequence。当前 K_n=4、K_a=16，因此每个样本有 21 个 tokens。序列第一位是目标节点属性，随后是 normal reference attributes，最后是 anomaly-side reference attributes。

DualRef token sequence

这个序列被输入 VecGAD 的 TransformerEncoder。Transformer 的作用是让 target token 在 normal references 与 anomaly-side references 的上下文中被编码，得到 reference-conditioned representation h_i。换言之，h_i 不是孤立节点 embedding，而是“目标节点在其 dual-reference context 下”的表示。

取 target token 的 encoder output 作为节点表示

这里下标 0 表示取序列中 target token 对应的输出。正常参考和异常侧参考的 embedding 也可以由对应 reference 节点的 encoded representations 获得，并在 readout 阶段求均值。

七、Directional Margin：为什么 cos(h-r_n, r_a-r_n) 是异常分数

在 encoder 输出空间中，我们为每个目标节点定义 normal reference center 与 anomaly-side reference center：

正常参考中心与异常侧参考中心

然后构造两个向量。第一个向量 u_i 是目标节点相对正常参考中心的偏移；第二个向量 d_i 是从正常参考中心指向异常侧参考中心的方向：

u_i=h_i-r_n(i),\qquad d_i=r_a(i)-r_n(i)

最终异常分数定义为二者的 cosine similarity：

Reference-Conditioned Directional Margin

这个公式的解释是：如果目标节点相对其局部正常参考的偏移方向，与“从正常参考到异常侧参考”的方向一致，那么它更可能位于异常侧，因此 score 较高。反之，如果目标节点虽然远离正常参考，但偏移方向并不朝向 anomaly-side reference center，则它不应被赋予同样高的异常分数。

Insight：tokenization 的关键不是“把邻居放进 Transformer”，而是通过一套半监督合法的 reference generation 规则，把每个节点放入一个局部 normal-to-anomaly coordinate system。Descriptor 决定 reference 的相似性语境，normal model 决定 anomaly-side prior，DualRef token sequence 决定 Transformer 的上下文，directional margin 决定最终解释性读出。

Key Finding：当前 AUC≈0.79 方案可以被完整表述为：construct local normal anchors and anomaly-side anchors without anomaly supervision, encode the target under this dual-reference context, and score whether the target displacement from the normal anchor aligns with the induced anomaly-side direction.

八、实验细节与结果

当前复现结果来自 Elliptic seed0，训练集合为 5% normal-only split。配置固定为 descriptor_mode=hop_attr、gn_mode=label_gate、ga_mode=normal_soft_or、normal_k=4、anom_k=16。encoder 使用 VecGAD TransformerEncoder，embedding_dim=256，GT_num_layers=3，GT_num_heads=2，dropout=0.4。closed-form margin 结果为 AUC=0.793789，AP=0.551031，top1% anomaly ratio=0.995475，top5% anomaly ratio=0.767978。

reference purity 诊断显示，normal references 的正常比例为 1.0；anomaly-side references 中真实异常比例整体约 0.1074，在真实异常节点上的 anomaly-side reference anomaly ratio 约 0.6837。这个结果说明 anomaly-side references 不是全局纯异常集合，但对真实异常节点而言，它们显著富集异常侧信号。这与方法设计一致：R_a(i) 不是标签监督的异常集合，而是一个由 normal rejection 与局部相似性诱导出来的异常方向。

Recommendation：后续写作中，应把 tokenization/reference generation 作为方法核心而不是实验细节。尤其要明确 anomaly-side references 的合法性：它们由正常分布偏离程度与局部相似性生成，不使用真实异常标签，因此符合半监督设定。这样读者才能理解为什么最终 cosine margin 不是任意手工分数，而是一个由 DualRef tokenization 诱导出的方向异常度量。
