Likelihood-Ratio with d-Conditioning: 如何在不生成伪异常的情况下利用 anomaly direction

Sender: Nexus | Scenario: DualRefGAD learnable head / semi-supervised normal-only objective | Date: 2026-05-12

Summary: The previous density probe failed because it treated anomaly direction d=r_a-r_n as a latent variable, discarding DualRef's core prior. This email proposes a more elegant approach: Likelihood-Ratio with d-Conditioning. The core idea is to still train only on normal nodes, but let anomaly direction d serve as conditioning variable; anomaly score becomes likelihood ratio log p(phi|d-like-context) - log p(phi|normal-context). This does not require explicit pseudo-anomaly generation nor stitching margin, instead using conditioning to let the model implicitly learn that "phi along d direction should have lower probability".

I. Core Idea: From Unconditional Density to Conditional Density + Ratio

The previous density probe objective was:

max_theta sum_{i in normal} log p_theta(phi_i)

This only requires normal relation features to have high probability under the model, but completely ignores anomaly direction d. Result: AUC only 0.667, far below margin 0.7938.

The key problem: d itself is constructed from anomaly-side reference, carrying information about "where anomalies should deviate". Treating it as ordinary latent variable means trying to infer anomaly direction from normal samples, which is the wrong information source.

Likelihood-ratio with d-conditioning corrects this:

s_i = log p_theta(phi_i | c_normal) - log p_theta(phi_i | c_anom)

Both conditioning contexts are constructed from anomaly direction d statistics, representing "normal-side" and "anomaly-side" assumptions. Model still trains on normal-only density, but anomaly score comes from likelihood ratio under two conditions.

Key Judgment: Likelihood-ratio approach does not generate pseudo-anomalies, does not stitch margin, still trains only on normal nodes, but lets anomaly direction d enter scoring through "hypothesis testing" via conditioning. This respects DualRef geometry better than pure density, and is more elegant than margin calibration.

II. Why Likelihood Ratio is More Reasonable Than Pure Density

Statistically, likelihood ratio test is the classic framework for "which hypothesis fits data better". In normal-only semi-supervised GAD, our information boundary is: only know which nodes are normal, not the true anomaly distribution. Likelihood ratio formulation:

  • Hypothesis H0 (normal assumption): Node's relation feature phi comes from normal context: phi ~ p(phi | c_normal).
  • Hypothesis H1 (anomaly assumption): Node's relation feature phi comes from anomaly-side context: phi ~ p(phi | c_anom).

Training uses only labeled normal nodes, so we only learn density function p(phi | context); during inference, compute likelihood under both hypotheses for each node, high ratio means more consistent with anomaly hypothesis.

III. How to Design Conditioning Variable

Option A: Use ||d|| as scalar conditioning

Simplest approach - use d's length as scalar conditioning:

c_i = ||d_i|| = ||r_a(i) - r_n(i)||

Train conditional density p(phi | ||d||). Inference:

s_i = log p(phi_i | ||d||_large) - log p(phi_i | ||d||_small)

||d||_large and ||d||_small can be fixed thresholds or quantiles from labeled normal nodes.

Pros: Simple, only scalar conditioning. Cons: ||d|| only captures length, not direction.

Option B: Use normalized d as conditioning vector

c_i = normalize(d_i)

Train p(phi | d_norm). Inference:

s_i = log p(phi_i | d_i_norm) - log p(phi_i | d_avg_norm)

Pros: Full direction information. Cons: 256-dim conditioning, need complex model.

Option C: Use anomaly-side rejection score

DualRef's existing anomaly-side rejection gives a scalar score:

c_i = g_a(i) from ga_mode=normal_soft_or

Pros: Already computed in DualRef pipeline. Cons: Coupled with other modules.

Recommendation for First Experiment

Use Option A: ||d|| as scalar conditioning for first diagnostic:

  • Simplest, avoids high-dim conditioning complexity.
  • ||d|| has physical interpretation: anomaly reference distance from normal reference.
  • Easy to implement: scalar feature into conditional flow.
  • After validation, can extend to vector conditioning.

IV. Concrete Experiment Design

Input Features

Use same relation features as previous density probe:

phi_i = concat(
    normalize(u_i),
    normalize(d_i),
    normalize(u_i) * normalize(d_i),
    |normalize(u_i) - normalize(d_i)|
)
c_i = ||d_i||

Model

Conditional RealNVP flow, input phi, conditioning c=||d||:

p_theta(phi_i | c_i) where c_i = ||d_i||

Train on labeled normal nodes only:

max_theta sum_{i in normal} log p_theta(phi_i | ||d_i||)

Inference Likelihood Ratio

For each node compute likelihood under two conditions:

s_i = log p_theta(phi_i | c_high) - log p_theta(phi_i | c_low)

Recommended strategy: per-node c vs normal mean

s_i = log p(phi_i | ||d_i||) - log p(phi_i | ||d||_normal_mean)

This tests: "my anomaly direction length represents anomaly degree" vs "I am like typical normal node's anomaly direction length".
