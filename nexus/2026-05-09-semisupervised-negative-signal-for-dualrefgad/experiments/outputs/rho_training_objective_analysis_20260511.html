RHO 训练目标分析与借鉴可能性：Deep One-class + Contrastive Alignment vs DualRef relation geometry

Nexus ｜ 文献调研与方案对比 ｜ 2026-05-11

摘要：通过阅读 RHO 论文，我发现其核心训练目标是 Deep One-class Classification + Cross-view Contrastive Alignment，不使用 pseudo-anomaly candidates。这与我们之前讨论的三类候选方案有明显 difference。RHO 的 contrastive alignment 思想可以借鉴，但它的 global center-based design 与 DualRefGAD 的 local reference-conditioned geometry 有本质差异。我建议重点借鉴 RHO 的多视图一致性思想，将其改造为“relation vectors vs normal reference center 的 contrastive alignment”，同时保持 DualRef 的局部化 reference selection。此外，建议补充读 GGAD（同一作者团队）和更广泛的 semi-supervised GAD survey，以形成更完整的文献判断。

一、RHO 训练目标的核心设计

RHO 的总 loss 由两部分组成：

RHO 总损失函数

其中 L_ccr 和 L_cwr 是跨通道与通道独立两个视图上的一类分类损失，L_GNA 是 Graph Normality Alignment 的对比损失。

一类分类损失：将正常节点压向超球体中心

L_{ccr} = \frac{1}{|V_l|} \sum_{i \in V_l} \| h_{ccr}^{(T)}(i) - c_{ccr} \|^2 + \sum_{t=1}^T \| W_{ccr}^{(t)} \|_F^2

这个 loss 的 intuition 是：

  • 只在标记正常节点 V_l 上训练；
  • 让正常节点的嵌入聚集到一个全局中心 c；
  • 异常节点在训练时不出现，但因为它们偏离正常分布，推理时自然距离中心更远。

对比对齐损失：跨视图一致性

GNA contrastive alignment (InfoNCE variant)

这个 loss 的 intuition 是：

  • 同一个节点在两个视图（ccr 和 cwr）上的表示应该一致；
  • 不同节点之间应该拉开距离；
  • 这个对比 loss 在所有节点（包括无标签节点）上计算，是一种自监督机制。

Insight：RHO 的核心 intuition 是不生成 pseudo-anomaly，而是通过学习正常分布的鲁棒边界，让异常自然落在边界外侧。这与方案 A/C 的“显式对比 pseudo-anomaly”有本质 difference。

三、DualRefGAD 与 RHO 的本质差异

维度RHODualRefGAD差异含义
Reference type全局超球体中心 c局部 normal/anomaly-side refs R_n(v), R_a(v)RHO global，DualRef local
Score geometry距离全局中心Directional margin cos(u,d)RHO unidirectional，DualRef bidirectional
Training lossOne-class + Contrastive alignment当前是 margin regression（诊断性）需要设计新 loss

关键判断：DualRefGAD 已经有一个完整的 local reference-conditioned coordinate system。RHO 的 global center-based one-class design 与 DualRef 的设计理念不同。直接把 RHO 的 loss 套到 DualRef 上，会丢弃 local reference 的解释性和几何优势。

四、可以借鉴的是什么

  1. Contrastive alignment 思想：RHO 的 GNA loss 让同一节点在两个视图上对齐。DualRefGAD 可以借鉴为：让 normal nodes 的 relation vectors u = h - R_n 对齐到 normal reference center，让 pseudo-anomaly candidates 的 relation vectors 更远离。
  2. 多视图一致性：RHO 用 ccr/cwr 两个视图防止过拟合。DualRefGAD 可以考虑：normal-side view 和 anomaly-side view 的 consistency constraint。
  3. Self-supervised alignment on all nodes：RHO 的对比 loss 在所有节点上计算，不只用 labeled normal。DualRefGAD 可以考虑：让 relation geometry 在全图范围内保持一致性。

五、建议的改造方向

结合 RHO 的 contrastive alignment 与 DualRef 的 relation geometry，我建议一个Hybrid Contrastive Normality Loss：

L_{normality} = -\log\frac{e^{\cos(u_i, r_n(i))/\tau}}{e^{\cos(u_i, r_n(i))/\tau} + \sum_{k \in R_a(i)} e^{\cos(u_i, u_k)/\tau}}

这个 loss 的 intuition 是：

  • 正常节点的 relation vector u_i 应该更接近其 normal reference center r_n(i)；
  • 同时应该远离 anomaly-side reference candidates R_a(i) 的 relation vectors；
  • 这保留了 DualRef 的局部化 reference，同时引入 RHO 的 contrastive alignment 思想。

六、文献补充建议

除了 RHO，建议补充以下文献：

  1. GGAD（同一作者团队）：GGAD 是 RHO 之前的工作，使用 pseudo-anomaly generation。对比 GGAD 与 RHO，可以理解“pseudo-anomaly vs no pseudo-anomaly”的取舍。
  2. VecGAD：已读过，是 DualRef 的 encoder 基础。
  3. SLAD /其他 semi-supervised GAD survey：了解该领域的主流设计模式。
  4. Deep One-class Classification for Graphs：如 ONE-DGC /其他图上一类分类方法，理解 global center-based 设计的完整生态。

当前判断：RHO 的 contrastive alignment 思想可以借鉴，但不能简单套用 global center-based loss。建议先读 GGAD，理解 pseudo-anomaly generation 的另一种路线，再决定是否采用 pairwise ranking 或 contrastive normality。

Recommendation：（1）先读 GGAD，理解同一作者团队从 pseudo-anomaly（GGAD）到 no pseudo-anomaly（RHO）的设计演变；（2）基于 DualRef 的 relation geometry，设计一个 relation-based contrastive normality loss，融合 RHO 的对比对齐思想；（3）如果 contrastive normality 在 tail ranking 上仍有问题，再考虑 pairwise ranking with pseudo-anomaly candidates。不要放弃 DualRef 的局部化 reference design，因为它是 DualRef 区别于 global center-based 方法的关键优势。
