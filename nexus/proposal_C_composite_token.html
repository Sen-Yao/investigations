🔮 方案 C: Composite Offset Token 详细报告

优先级：P3

工作量：中（3-5 天）

类型：架构改进，借鉴 NTFormer 的 Node2Par 思想

📋 核心思想

借鉴 NTFormer 的 Node2Par，将 Offset 作为多种 token 类型之一。

目标：

  • 从多个视角表达节点特征
  • Offset 与其他特征类型互补
  • 无需针对不同图类型修改模型

📐 NTFormer 的 Node2Par 思想

核心问题

现有方法只使用单一类型的 token 生成，只能表达部分图信息。

Node2Par 解决方案

对于每个节点，使用多种 token 元素构建 token 序列：

Token 类型 A: 节点自身特征
Token 类型 B: 邻居特征聚合
Token 类型 C: 属性特征
Token 类型 D: 结构特征

→ 组合生成多种 token 序列
→ 全面表达节点特征
→ 无需图特定修改

关键优势

  • 多视角：从不同角度表达节点
  • 通用性：适配各种图类型
  • 灵活性：可选择合适的 token 类型

🔬 Composite Offset Token 设计

Token 类型设计

Token 类型内容维度作用
Original原始 hop 特征[K+1, D]保留原始信息
Offsethop_k - hop_0[K, D]捕获变化方向
Deltahop_k - hop_{k-1}[K, D]捕获增量变化
Degree节点度特征[1, D']结构信息
Position位置编码[K+1, D]位置信息

方案 C1: Concatenation（拼接）

思路

将多种 token 类型拼接成更长的序列。

实现

def composite_token_concatenation(node, features, adj):
    """
    拼接式组合 token
    """
    # 1. 原始 hop 特征
    original_tokens = sample_hop_features(node, features, adj)  # [K+1, D]
    
    # 2. Offset token
    offset_tokens = original_tokens[1:] - original_tokens[0:1]  # [K, D]
    
    # 3. Delta token
    delta_tokens = original_tokens[1:] - original_tokens[:-1]  # [K, D]
    
    # 4. 度特征
    degree_token = compute_degree_features(node, adj)  # [1, D']
    
    # 5. 拼接
    composite_tokens = torch.cat([
        original_tokens,      # [K+1, D]
        offset_tokens,        # [K, D]
        delta_tokens,         # [K, D]
        degree_token.expand(K, -1)  # [K, D']
    ], dim=0)  # 总长度: (K+1) + K + K + K = 3K + 1
    
    return composite_tokens

特点

  • 简单直接：易于实现
  • 信息完整：保留所有信息
  • 缺点：序列变长，计算量增加

方案 C2: Multi-Channel（多通道）

思路

每种 token 类型作为独立通道，分别处理后再融合。

实现

class MultiChannelTokenEncoder(nn.Module):
    def __init__(self, d_model, num_types=3):
        super().__init__()
        self.num_types = num_types
        
        # 每个 token 类型有独立的编码器
        self.original_encoder = TransformerEncoder(d_model)
        self.offset_encoder = TransformerEncoder(d_model)
        self.delta_encoder = TransformerEncoder(d_model)
        
        # 融合层
        self.fusion = nn.Linear(d_model * num_types, d_model)
    
    def forward(self, original_tokens, offset_tokens, delta_tokens):
        """
        多通道处理
        """
        # 独立编码
        original_repr = self.original_encoder(original_tokens)  # [B, K+1, D]
        offset_repr = self.offset_encoder(offset_tokens)         # [B, K, D]
        delta_repr = self.delta_encoder(delta_tokens)            # [B, K, D]
        
        # 池化为节点表示
        original_node = original_repr.mean(dim=1)  # [B, D]
        offset_node = offset_repr.mean(dim=1)      # [B, D]
        delta_node = delta_repr.mean(dim=1)        # [B, D]
        
        # 融合
        concat_repr = torch.cat([original_node, offset_node, delta_node], dim=-1)
        node_repr = self.fusion(concat_repr)
        
        return node_repr

特点

  • 独立建模：每种类型有专门编码器
  • 灵活融合：可学习的融合权重
  • 缺点：参数量增加

方案 C3: Gated Combination（门控组合）

思路

使用门控机制自适应选择不同 token 类型的重要性。

实现

class GatedTokenCombination(nn.Module):
    def __init__(self, d_model, num_types=3):
        super().__init__()
        self.gate = nn.Sequential(
            nn.Linear(d_model * num_types, d_model),
            nn.Sigmoid()
        )
        self.projection = nn.Linear(d_model * num_types, d_model)
    
    def forward(self, original_repr, offset_repr, delta_repr):
        """
        门控组合
        """
        # 拼接所有表示
        concat = torch.cat([original_repr, offset_repr, delta_repr], dim=-1)
        
        # 计算门控权重
        gate_weights = self.gate(concat)  # [B, D]
        
        # 加权投影
        combined = self.projection(concat) * gate_weights
        
        return combined

特点

  • 自适应：自动学习重要性
  • 可解释：门控权重反映类型重要性
  • 适合异常检测：不同数据集可能偏好不同类型
