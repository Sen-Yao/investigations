# BypassVoxG 文献调研

## 核心发现

### 1. Bypass Connection 原理

**ResNet 核心公式**：
$$y = \mathcal{F}(x) + x$$

**为什么能保留信息**：
- 前向传播：原始输入无损传递到深层
- 反向传播：梯度通过恒等映射回传，防止梯度消失

---

### 2. 降维的信息损失

**关键洞察**：
> 线性投影会丢失位于 $W$ 零空间的信息。**异常样本往往在正交的残差子空间中具有显著投影**。

**PCA 的问题**：
- 主成分 = 正常模式（方差最大）
- 次要成分 = 异常信号（方差小但重要）

---

### 3. 双路径架构设计

| 路径 | 功能 | 设计 |
|------|------|------|
| **High-Res Path** | 保留高频细节 | 不降维，稀疏计算 |
| **Low-Res Path** | 捕获全局语义 | 标准 Transformer/GNN |

**HRNet 关键设计**：始终保持高分辨率路径

---

### 4. 异常检测中的陷阱

**重构网络的恒等映射问题**：
- Skip Connection 可能导致异常被完美重构
- 解决方案：Masked Modeling (MAE)

---

### 5. BypassVoxG 设计建议

#### 架构

```
Hop Features [N, 7, 745]
         │
    ┌────┴────┐
    │         │
    ▼         ▼
┌───────┐  ┌──────────────┐
│Trans- │  │  Bypass      │
│former │  │  (745维直接)  │
│(128维)│  │              │
└───┬───┘  └──────┬───────┘
    │             │
    └──────┬──────┘
           │
           ▼
    ┌──────────────┐
    │ 门控融合      │
    │ G = σ(W1*T +  │
    │       W2*B)  │
    └──────────────┘
```

#### 融合公式

$$H_{out} = G \odot H_{trans} + (1-G) \odot H_{bypass}$$

## 设计决策

### 决策 1：不使用简单相加融合

**原因**：高维稀疏特征和低维稠密特征分布不同

**选择**：门控融合（Gated Fusion）

---

### 决策 2：引入 Initial Residual

**公式**：
$$H^{(l+1)} = (1-\alpha) \text{Transformer}(H^{(l)}) + \alpha H^{(0)}$$

**作用**：防止异常特征被稀释

---

### 决策 3：避免恒等映射泄漏

**方法**：重构目标不直接使用 Bypass 特征

**替代**：Bypass 特征用于异常分数计算，而非重构

## 关键发现

### 发现 1：Bypass 通道不如 concat

**原因**：
1. 信息丢失：压缩维度后丢失关键特征
2. 梯度干扰：去噪/离群点生成的梯度干扰主任务
3. 特征未对齐：融合时特征不在同一语义空间

### 发现 2：concat 是最优基础架构

**原因**：
1. 无损：保留所有原始信息
2. 简单：让网络自己学习重要性
3. 高效：梯度回传直接清晰
