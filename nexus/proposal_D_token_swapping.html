📋 核心思想

借鉴 SwapGT，利用 Offset 方向的语义相关性进行 token swapping。

目标：

  • 增强 Offset 表示的多样性
  • 利用全局语义信息
  • 提升模型泛化能力

📐 SwapGT 的 Token Swapping 思想

核心问题

现有 tokenized GT 只关注一阶邻居，生成的 token 序列多样性不足。

SwapGT 解决方案

Token Swapping 操作：

如果 node_i 和 node_j 语义相似（如在同一类别中）：
  可以交换它们某些位置的 token
  例如：swap(node_i.hop_2, node_j.hop_2)

效果：
  - 生成更多样化的 token 序列
  - 增强模型泛化能力
  - 利用全局语义信息

关键创新

  • 语义驱动：基于语义相似性而非随机交换
  • 数据增强：Token swapping 作为数据增强
  • Center Alignment Loss：约束多个 token 序列的表示学习

🔬 Offset-based Token Swapping 设计

核心问题

如何定义节点间的"Offset 语义相似性"？

Offset 相似性定义

定义 1: 方向相似性

def direction_similarity(offset_i, offset_j):
    """
    基于 Offset 方向的相似性
    """
    # 归一化为方向向量
    dir_i = offset_i / (||offset_i|| + eps)
    dir_j = offset_j / (||offset_j|| + eps)
    
    # 平均方向相似度（跨所有 hop）
    sim = mean(cosine_similarity(dir_i, dir_j, dim=-1))
    
    return sim

定义 2: 变化模式相似性

def pattern_similarity(offset_i, offset_j):
    """
    基于 Offset 变化模式的相似性
    """
    # Offset 的变化趋势
    trend_i = offset_i[1:] - offset_i[:-1]  # Offset 的差分
    trend_j = offset_j[1:] - offset_j[:-1]
    
    # 趋势相似度
    sim = cosine_similarity(trend_i.flatten(), trend_j.flatten())
    
    return sim

定义 3: 结构感知相似性

def structure_aware_similarity(node_i, node_j, adj):
    """
    结合结构信息的相似性
    """
    # Offset 相似性
    offset_sim = direction_similarity(node_i.offset, node_j.offset)
    
    # 结构相似性（如共同邻居比例）
    neighbors_i = set(adj[node_i].nonzero())
    neighbors_j = set(adj[node_j].nonzero())
    jaccard = len(neighbors_i & neighbors_j) / len(neighbors_i | neighbors_j)
    
    # 加权组合
    combined_sim = 0.7 * offset_sim + 0.3 * jaccard
    
    return combined_sim

🔬 Token Swapping 策略

策略 D1: 随机 Swapping（基线）

思路

随机选择节点对进行 token swapping。

实现

def random_swapping(tokens, swap_prob=0.1):
    """
    随机 token swapping
    """
    N, K, D = tokens.shape
    swapped_tokens = tokens.clone()
    
    for i in range(N):
        if random.random() < swap_prob:
            # 随机选择另一个节点
            j = random.randint(0, N-1)
            while j == i:
                j = random.randint(0, N-1)
            
            # 随机选择一个 hop 位置
            hop = random.randint(0, K-1)
            
            # 交换
            swapped_tokens[i, hop], swapped_tokens[j, hop] = \
                swapped_tokens[j, hop].clone(), swapped_tokens[i, hop].clone()
    
    return swapped_tokens

特点

  • 简单：无需计算相似性
  • 缺点：可能交换语义不相关的节点

策略 D2: Offset-similarity Swapping（推荐）

思路

只交换 Offset 方向相似的节点。

实现

class OffsetSimilaritySwapping:
    def __init__(self, similarity_threshold=0.8):
        self.threshold = similarity_threshold
    
    def compute_similarity_matrix(self, offsets):
        """
        计算所有节点对的 Offset 相似性
        """
        N = len(offsets)
        sim_matrix = torch.zeros(N, N)
        
        for i in range(N):
            for j in range(i+1, N):
                sim = direction_similarity(offsets[i], offsets[j])
                sim_matrix[i, j] = sim
                sim_matrix[j, i] = sim
        
        return sim_matrix
    
    def swap(self, tokens, offsets, hop_indices):
        """
        基于 Offset 相似性的 token swapping
        """
        sim_matrix = self.compute_similarity_matrix(offsets)
        swapped_tokens = tokens.clone()
        
        N, K, D = tokens.shape
        
        for i in range(N):
            # 找到与 i 相似的节点
            similar_nodes = (sim_matrix[i] > self.threshold).nonzero().squeeze()
            
            if len(similar_nodes) > 0:
                # 随机选择一个相似节点
                j = similar_nodes[random.randint(0, len(similar_nodes)-1)]
                
                # 随机选择一个 hop 位置
                hop = random.choice(hop_indices)
                
                # 交换
                swapped_tokens[i, hop], swapped_tokens[j, hop] = \
                    swapped_tokens[j, hop].clone(), swapped_tokens[i, hop].clone()
        
        return swapped_tokens

特点

  • 语义保持：交换语义相似的节点
  • 多样性：增强表示多样性
  • 缺点：需要计算相似性矩阵

策略 D3: Class-aware Swapping

思路

利用标签信息，只在同类别节点间进行 swapping。

实现

class ClassAwareSwapping:
    def __init__(self):
        pass
    
    def swap(self, tokens, labels, hop_indices):
        """
        类别感知的 token swapping
        """
        swapped_tokens = tokens.clone()
        N, K, D = tokens.shape
        
        # 按类别分组
        unique_labels = labels.unique()
        
        for label in unique_labels:
            # 同一类别的节点
            same_class = (labels == label).nonzero().squeeze()
            
            if len(same_class) > 1:
                # 随机配对
                perm = torch.randperm(len(same_class))
                
                for idx in range(0, len(perm)-1, 2):
                    i = same_class[perm[idx]]
                    j = same_class[perm[idx+1]]
                    
                    # 随机选择 hop
                    hop = random.choice(hop_indices)
                    
                    # 交换
                    swapped_tokens[i, hop], swapped_tokens[j, hop] = \
                        swapped_tokens[j, hop].clone(), swapped_tokens[i, hop].clone()
        
        return swapped_tokens

特点

  • 类别保持：交换后标签不变
  • 适合监督/半监督：利用标签信息

📊 Center Alignment Loss

问题

Token swapping 后，同一个节点有多种 token 序列表示，如何保证表示一致性？

SwapGT 的解决方案：Center Alignment Loss

def center_alignment_loss(repr_original, repr_swapped):
    """
    中心对齐损失
    
    目标：原始序列和交换序列的表示应该接近
    """
    # 计算中心表示
    center = (repr_original + repr_swapped) / 2
    
    # 两个表示都应该接近中心
    loss = F.mse_loss(repr_original, center) + F.mse_loss(repr_swapped, center)
    
    return loss

作用

  • 一致性：保证 swapping 不破坏表示
  • 稳定性：稳定训练过程

⚠️ 潜在问题

问题 1: 计算开销

计算 Offset 相似性矩阵需要 O(N²) 复杂度。

解决方案：采样部分节点计算相似性，或使用近似方法。

问题 2: Swapping 破坏信息

不当的 swapping 可能破坏有用的信息。

解决方案：只交换语义相似的节点，并使用 alignment loss 约束。

问题 3: 与异常检测任务的适配

SwapGT 针对节点分类，异常检测可能需要不同策略。

解决方案：可能需要在正常节点内部 swapping，避免正常-异常间的 swapping。
