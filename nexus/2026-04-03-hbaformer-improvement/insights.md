# 发现和洞见 - HBAFormer 改进

## 核心发现

### 1. Attention Bias 干扰异常感知
**来源**: GLM-5.1 辩论 (2026-04-03 18:25)

**洞见**: 
- Hop-based Attention Bias 是"蒙住模型眼睛"- 可能洗掉异常节点的特征偏离信号
- 正常节点主导注意力，异常信号被压制

**实验证据**: v1 (0.7943) < v2 (0.7749) < VecGAD (0.8960)

---

### 2. B1 (乘法缩放) 最优的原因
**来源**: GLM-5.1 辩论 (2026-04-03 19:28)

**三个原因**:
1. **信噪比调节器**: 直接告诉模型"这个 Hop 应被信任多少"2. **不增加维度负担**: B2/B3 拼接扩展维度，增加优化难度
3. **与注意力兼容**: 形成"双重乘法门控"- 注意力决定"关注哪个token"- 缩放决定"信号强度"---

### 3. GLM-5.1 预测错误的根本原因
**来源**: GLM-5.1 辩论 (2026-04-03 19:28)

| 错误 | 说明 |
|------|------|
| 混淆信息量与利用效率 | B2 给了最多信息，但模型无法高效利用 |
| 低估归纳偏置力量 | B1 包含最强先验，在半监督场景极有效 |
| 忽视优化难度差异 | B2/B3 需从零学习 Hop 语义，B1 直接给出解 |

**教训**: 在低监督场景下，强归纳偏置 > 更多自由度

---

### 4. Photo 特殊性修正理解
**来源**: GLM-5.1 辩论 (2026-04-03 19:28)

**修正**: "特征偏离为主" ≠ "结构信息无用"- Hop 提供"不同距离下的特征聚合视图"- 异常节点的邻居聚合也会偏离（特征污染邻居表示）
- B1 相当于"多尺度特征融合"---

## 下一步建议

### 方向 B：Hop-guided Smoothing Loss + b1 缩放
**来源**: GLM-5.1 辩论 (2026-04-03 19:28)

**核心思想**:
- 保留 v3-b1 的乘法缩放
- 新增 Hop 平滑性损失作为辅助训练信号

**公式**:
```python
L_smooth = Σ_k ||z_k - z_{k+1}||² / σ_k²
# 正常节点：Hop 序列平滑
# 异常节点：Hop 序列不连续
```

**验证方案**:
- 固定 hop_variant=b1
- Sweep smooth_loss_weight ∈ {0, 0.01, 0.05, 0.1, 0.2, 0.5}
- 5 seeds × 6 weights = 30 runs
