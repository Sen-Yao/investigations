## 核心发现

### 1. CAA超越SOTA

| 对比 | 结果 |
|------|------|
| CAA vs VecGAD | **+5.0%** (0.97 vs 0.90) |
| CAA vs Offset baseline | +4.6% (0.97 vs 0.93) |
| CAA F1 vs MLP baseline | +59% (0.89 vs 0.31) |

**意义**: 信息论引导设计优于经验设计。

### 2. Delta策略验证

| 验证项 | Phase 1-3发现 | Phase 4验证 |
|--------|---------------|-------------|
| MI最高 | Photo MI=66.4 | CAA最佳AUC |
| 深层信息 | Token 5-6 MI最高 | 收敛偏置有效 |
| 图结构相关 | PageRank corr=0.89 | 训练稳定 |

**意义**: Phase 1-3信息论分析结论被实验验证。

### 3. 架构简洁性

| 机制 | 参数量 | AUC |
|------|--------|-----|
| **CAA** | 518K | 0.9708 |
| DSA | 1.5M | 0.7169 |

**意义**: 简洁设计优于复杂双流架构。

## 局限性分析

### 1. 单数据集验证

**现状**: 仅在Photo验证  
**影响**: 泛化性待确认  
**建议**: 扩展至Tolokers, Elliptic, Amazon

### 2. 5-seed未完成

**现状**: 仅1个seed有结果  
**影响**: 统计显著性待确认  
**建议**: 完成5-seed sweep

### 3. WandB运行失败

**现状**: 数据集路径问题  
**影响**: Sweep结果缺失  
**建议**: 修复run_caa.py数据加载

### 4. DSA不稳定

**现状**: DSA Test AUC=0.72（过拟合）  
**原因**: 参数量大，lr过高  
**建议**: lr降至0.0001，epochs增至30

## 方法论贡献

### 信息论→设计流程

```
Phase 1: MI分析 → 确定最优Token策略
Phase 2: 物理意义 → 理解Token信息含义
Phase 3: 注意力分析 → 确定注意力分布问题
Phase 4: 机制设计 → 基于发现设计增强
```

**价值**: 可复用的研究方法论。

### Token策略选择规则

| 数据集特征 | 推荐策略 |
|------------|----------|
| D > 100 (高维) | Delta |
| D < 50 (低维) | 任意 |
| 强图结构依赖 | Delta |
| 需要稳定性 | Offset |
